# Аналіз утримання користувачів (Cohort Analysis) 

## Опис проекту
Цей скрипт призначений для проведення **когортного аналізу** активності користувачів OTT-платформи за 2015 рік. Він дозволяє відстежити, як довго користувачі залишаються активними після реєстрації (Retention Rate).

## Технології та методи
- **MySQL**: використання CTE (Common Table Expressions) та складних агрегацій.
- **Сегментація**: дані розбиті за брендами (`brand_1`, `brand_2`), країнами, мовами та типами пристроїв.

## Структура вихідних даних
Результатом роботи скрипта є матриця утримання:
- `cohort`: місяць реєстрації користувачів (когорта).
- `brand`: назва бренду сервісу.
- `country`, `lang`, `deviceTitle`: розрізи для глибинного аналізу.
- `_0, _1, ... _11`: кількість унікальних користувачів, активних через N місяців після реєстрації.

## Візуалізація та Дашборд
Результати цього запиту використовуються як джерело даних (Backend) для інтерактивного дашборда.
- **Метрики у %**: Дашборд автоматично розраховує Retention % для кожного місяця відносно нульового (`_0`).
- **Heatmap**: Візуалізація у вигляді теплової карти дозволяє миттєво ідентифікувати аномальний відтік користувачів.

## Бізнес-цінність
Аналіз допомагає відповісти на ключові питання:
1. **Churn Rate**: На якому місяці ми втрачаємо найбільше користувачів?
2. **LTV**: Який середній "термін життя" користувача в розрізі різних пристроїв?
3. **Ефективність маркетингу**: Які когорти (місяці реєстрації) виявилися найбільш лояльними?

## Як використовувати
Скрипт розрахований на роботу з помісячно секціонованими таблицями логів (`userLog_MM_YYYY`).
